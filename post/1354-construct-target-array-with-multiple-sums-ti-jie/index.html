1354. Construct Target Array With Multiple Sums —— 题解

2020-03-12 LeetCode ， Algorithm

本题作为Contest 176的压轴题，难度还是有一些，不过比起普通的算法题，脑筋急转弯的含量略大一点。只要考虑对方向，做出来还是不算太难。本人在Contest时没做出来，结束了才想到方向错了...
附上题目链接 Construct Target Array With Multiple Sums😁

题目描述

给定一个数组，size不一定，但是值都是1
然后我们可以选定其中的一个值，把它设置成整个数组的和，如
[1, 1, 1] -> [1, 1, 3] -> [1, 1, 5] -> [1, 7, 5] ... 无限处理下去
然后给出一个数组，问能否通过上述内容构建出这个数组

题目分析

一个数组构建题目，直白的想法可以直接DFS处理所有可能，然后与原数组判断，但这种方法时间复杂度是O(n^n)，是一个每个结点都有n个儿子的树，即便有一定剪枝操作，这个时间复杂度也是无法接受的。

Contest时一直在考虑这个剪枝优化... 白白浪费时间，应该冷静冷静重新思考

如果正向构建不行，那么我们可以看反向减少能否做到。
我们知道给定数组肯定有一个最大值，如[9, 3, 5]其中9是最大值，而当前这个数组的最大值是上一次数组的和，因此在这次操作前，可以用最大值 减去 其他所有值得和，即该数组的上一个状态
[9, 3, 5] -> [1, 3, 5] -> [1, 3, 1] -> [1, 1, 1]
循环这个过程，直到我们能得到所有值均为1的数组（如果提前得到小于0的值，可以认为不合法而直接return false）

解法一 反向构建

fun isPossible(target: IntArray): Boolean {
    val pq = PriorityQueue<Long>(compareByDescending { it })
    var sum = 0L
    target.forEach {
        pq.offer(it.toLong())
        sum += it
    }
    while (pq.peek() != 1L) {
        val item = pq.poll()
        if (item == 1L) {
            continue
        }
        val last = sum - item
        if (item - last < 1) {
            return false
        }
        pq.offer(item - last)
        sum -= last
    }
    return true
}

利用反向的优先级队列，把最大值放在最前面，然后让最大值 减去 剩余的 值，把结果塞回队列，重复这个过程，不断更新最大值，直到最大值为1 or 塞入的值 < 1

这个算法还有改进的空间，请看解法二

解法二 反向构建 + mod优化

上述Code展示了很优雅的解法，不过在一些特殊值得场景下，时间复杂度依然很高。
如 [1999, 1] 这种数组，每次都会让较大的值减去1，但是要减去1998次才能达到终点，而这可以优化
最大值更新后 如果仍是最大值，那么其实剩余值并没有变化，每次都减去一个值，直到自己小于它，这不就是取余操作么！优化后的代码如下：

val pq = PriorityQueue<Long>(compareByDescending { it })
var sum = 0L
target.forEach {
    pq.offer(it.toLong())
    sum += it
}
while (true) {
    val item = pq.poll() ?: 0L
    sum -= item
    // 最大值为1，则其余数字都是1
    // other sum 为1，则当前值可以直接化为1
    if (item == 1L || sum == 1L) {
        return true
    }
    // 当前值 小于等于 其他值的和，不合法
    if (item <= sum || sum == 0L) {
        return false
    }
    pq.add(item % sum)
    sum += item % sum
}
